# AI 智能项目管理平台 - 功能操作指南

## 前置条件

- 已运行 `npm run dev`，服务在 `http://localhost:3000`
- `.env.local` 中已填写真实的 `DEEPSEEK_API_KEY`
- 已执行 `npx prisma db seed` 写入种子数据

---

## 功能一：任务看板拖拽

**组件：** `TaskKanban`  
**说明：** 三列看板（待办 / 进行中 / 已完成），拖拽卡片即可更新任务状态，支持优先级标识和负责人展示。

**操作步骤：**
```
npm install
npx prisma migrate dev --name init
npm run db:seed
npm run dev
```
1. 打开 `http://localhost:3000`，自动跳转到项目列表
2. 点击「某型号发动机研发论证项目」卡片进入详情页
3. 切换到「任务」标签页，看到三列看板
4. 将「项目月度进度报告编制」从「进行中」列拖拽到「已完成」列
5. 状态自动保存到数据库，顶部进度条数字实时更新

---

## 功能二：AI 会议安排提醒

**组件：** `MeetingReminderBanner`  
**说明：** 检测到有「会议相关」任务未绑定会议记录时，详情页顶部自动出现 AI 提醒横幅，点击可用 AI 生成会议标题和议程。

**操作步骤：**

1. 在项目详情页顶部，看到蓝色提醒横幅（提示 2 个待安排会议）
2. 点击「安排会议」按钮，弹出创建会议对话框
3. AI 已预填好会议标题和议程（由 DeepSeek 生成）
4. 选择参会人员，设置会议时间，点击「创建会议」
5. 提醒横幅消失，任务与会议绑定完成

---

## 功能三：AI 解析会议纪要 → 生成待办

**组件：** `MinutesUploader`  
**说明：** 上传 `.txt` / `.md` 格式的会议纪要，AI 自动提取行动项，支持编辑后一键入库成为任务。

**操作步骤：**

1. 完成功能二，进入刚创建的会议详情页
2. 在本地新建 `minutes.txt` 文件，粘贴以下示例内容：

```
会议主题：发动机技术方案评审会议
时间：2024年6月20日
参会人：张伟、李娜、王磊、赵敏

会议决议：
1. 王磊负责在6月30日前完成压气机CFD仿真模型初稿，优先级高
2. 李娜负责在7月5日前整理推力指标论证数据，形成报告，优先级高
3. 赵敏负责在6月25日前更新项目进度看板并发送月报，优先级中
4. 张伟负责联系外部专家，7月10日前确认评审专家名单，优先级低
```

3. 将 `minutes.txt` 拖拽到页面上传区，或点击「选择文件」选取
4. AI 解析完成后，展示提取到的行动项列表（含负责人、截止日期、优先级）
5. 可编辑任意字段，确认无误后点击「确认入库」，批量创建为项目任务

## 常见问题

| 现象 | 原因 | 解决方法 |
|------|------|----------|
| 顶部提醒横幅未出现 | 种子数据未写入 | 执行 `npx prisma db seed` |
| AI 预填内容为空或报错 | API Key 未生效 | 检查 `.env.local` 中 `DEEPSEEK_API_KEY` 是否为真实 Key |
| 纪要解析失败 | 文件格式不对 | 仅支持 `.txt` 或 `.md`，不支持 Word/PDF |
| 拖拽后状态未保存 | 数据库未初始化 | 执行 `npx prisma migrate dev` |
